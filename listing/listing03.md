Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
Выведет:
  <nil>
  false
Чтобы проходить проверку на равенство с nil динамический тип интерфейсного
значения тоже должен быть nil. В данном же случае интерфейсное значение err
имеет динамический тип *fs.PathError.
Интерфейсное значение в общем виде можно представить в виде двух указателей:
первый указывает на информацию о типе, хранящемся в интерфейсе (если точнее то
на таблицу, хранящую указатель на тип и указатели на методы типа, реализующие
интерфейс), и второй указывает на связанные данные.
Любой тип в Go реализует пустой интерфейс. И первый указатель указывает
непостредственно на тип, а не таблицу (т.к. в пустом интерфейсе нет методов).

```
